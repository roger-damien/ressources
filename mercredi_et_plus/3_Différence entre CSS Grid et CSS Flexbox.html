Différence entre CSS Grid et CSS Flexbox

Posted on 5 de juillet de 2022by Jack Sparrow

Grid : CSS Grid Layout, est un système de mise en page basé sur une grille bidimensionnelle avec des lignes et des colonnes, facilitant la conception de pages Web sans avoir à utiliser de flotteurs et de positionnement. Comme les arrayx, la disposition en grille nous permet d’aligner les éléments en colonnes et en lignes.

Pour commencer, vous devez définir un élément conteneur en tant que grille avec display: grid, définir les tailles de colonne et de ligne avec grid-template-columns et grid-template-rows, puis placer ses éléments enfants dans la grille avec grid-column et ligne de grille.

Exemple:

HTML

<!DOCTYPE html>
<html lang="en">
<head>
    <style>
        .main{
              
            display: grid;
            display: grid; 
            grid: auto auto / auto auto auto auto; 
            grid-gap: 10px; 
            background-color: green; 
            padding: 10px; 
        }
        .gfg { 
            background-color: rgb(255, 255, 255); 
            text-align: center; 
            padding: 25px 0; 
            font-size: 30px; 
        } 
    </style>
</head>
<body>
    <h2 style="text-align: center;">Welcome To GeeksForGeeks </h2>
    <div class="main">
        <div class="gfg">Home</div>
        <div class="gfg">Read</div>
        <div class="gfg">Write</div>
        <div class="gfg">About Us</div>
        <div class="gfg">Contact Us</div>
        <div class="gfg">Privacy Policy</div>
    </div>
</body>
</html>

Flexbox : La CSS Flexbox propose une mise en page unidimensionnelle. Il est utile pour allouer et aligner l’espace entre les éléments d’un conteneur (constitué de grilles). Il fonctionne avec toutes sortes d’appareils d’affichage et de tailles d’écran.

Pour commencer, vous devez définir un élément conteneur comme une grille avec display : flex;

Exemple:

HTML

<!DOCTYPE html>
<html lang="en">
<head>
    <style>
        .main{
              
            display: flex;
            display: flex; 
            grid: auto auto / auto auto auto auto; 
            grid-gap: 10px; 
            background-color: green; 
            padding: 10px; 
        }
        .gfg { 
            background-color: rgb(255, 255, 255); 
            text-align: center; 
            padding: 25px 0; 
            font-size: 30px; 
        } 
    </style>
</head>
<body>
    <h2 style="text-align: center;">Welcome To GeeksForGeeks </h2>
    <div class="main">
        <div class="gfg">Home</div>
        <div class="gfg">Read</div>
        <div class="gfg">Write</div>
        <div class="gfg">About Us</div>
        <div class="gfg">Contact Us</div>
        <div class="gfg">Privacy Policy</div>
    </div>
</body>
</html>

Unicité dans Grid et Flexbox :

Une contre deux dimensions :

  • Grid est conçu pour une mise en page bidimensionnelle tandis que Flexbox en est un. Cela signifie que Flexbox peut fonctionner sur les lignes ou les colonnes à la fois, mais que Grids peut fonctionner sur les deux.
  • Flexbox, vous donne plus de flexibilité lorsque vous travaillez sur l’un ou l’autre élément (ligne ou colonne). Le balisage HTML et CSS seront faciles à gérer dans ce type de scénario.
  • GRID vous offre plus de flexibilité pour vous déplacer dans les blocs, quel que soit votre balisage HTML.

Contenu d’abord vs mise en page d’abord :

  • L’unicité majeure entre Flexbox et Grids est que le premier fonctionne sur le contenu tandis que le second est basé sur la mise en page.
  • La disposition Flexbox est la mieux adaptée aux composants d’application et aux dispositions à petite échelle, tandis que la disposition Grille est conçue pour les dispositions à plus grande échelle qui ne sont pas linéaires dans leur conception.

Différence entre Grid et Flexbox :

1. Dimensionnalité et flexibilité :

  • Flexbox offre un meilleur contrôle sur l’alignement et la répartition de l’espace entre les éléments. Étant unidimensionnel, Flexbox ne traite que des colonnes ou des lignes.
  • Grid a des capacités de mise en page en deux dimensions qui permettent des largeurs flexibles en tant qu’unité de longueur. Cela compense les limites de Flex.

2. Alignement :

  • Flex Direction permet aux développeurs d’aligner les éléments verticalement ou horizontalement, ce qui est utilisé lorsque les développeurs créent et inversent des lignes ou des colonnes.
  • CSS Grid déploie des unités de mesure fractionnaires pour la fluidité de la grille et la fonctionnalité de mot-clé automatique pour ajuster automatiquement les colonnes ou les lignes.

3. Gestion des articles

  • Flex Container est l’élément parent tandis que Flex Item représente les enfants. Le Flex Container peut assurer une représentation équilibrée en ajustant les dimensions des éléments. Cela permet aux développeurs de concevoir pour des tailles d’écran fluctuantes.
  • Grid prend en charge le placement de contenu implicite et explicite. Son automatisation intégrée lui permet d’étendre automatiquement les éléments de ligne et de copier les valeurs dans la nouvelle création à partir de l’élément précédent.

Conclusion

  • CSS Grids vous aide à créer la mise en page extérieure de la page Web. Vous pouvez créer une conception complexe et réactive avec cela. C’est pourquoi on l’appelle ‘mise en page d’abord’.
  • Flexbox aide principalement à aligner le contenu et à déplacer les blocs.
  • Les grilles CSS sont destinées aux mises en page 2D. Cela fonctionne avec les lignes et les colonnes.
  • Flexbox fonctionne mieux dans une seule dimension (lignes OU colonnes).
  • Ce sera plus rapide et utile si vous utilisez les deux en même temps.
